# 🤝 Contribuindo para o Trading Bot PRO 4.0

Obrigado por considerar contribuir! Este projeto é open source e contribuições são bem-vindas.

---

## 📋 Como Contribuir

### 1. Fork o Projeto
```bash
# Clique em "Fork" no GitHub
```

### 2. Clone seu Fork
```bash
git clone https://github.com/SEU_USUARIO/trading-bot-pro.git
cd trading-bot-pro
```

### 3. Crie uma Branch
```bash
git checkout -b feature/nova-feature
# ou
git checkout -b fix/correcao-bug
```

### 4. Faça suas Alterações
- Escreva código limpo e bem documentado
- Adicione comentários quando necessário
- Siga o estilo do código existente

### 5. Teste suas Alterações
```bash
# Teste em conta DEMO primeiro!
python main.py
```

### 6. Commit suas Mudanças
```bash
git add .
git commit -m "feat: Adiciona nova funcionalidade X"
# ou
git commit -m "fix: Corrige bug Y"
```

### 7. Push para seu Fork
```bash
git push origin feature/nova-feature
```

### 8. Abra um Pull Request
- Vá no GitHub e clique em "New Pull Request"
- Descreva suas mudanças claramente
- Explique por que a mudança é necessária

## 📝 Padrões de Código

### Python
- Use PEP 8
- Docstrings em funções públicas
- Type hints quando possível
- Nomes descritivos de variáveis

### Commits
Use Conventional Commits:
- `feat:` Nova funcionalidade
- `fix:` Correção de bug
- `docs:` Documentação
- `style:` Formatação
- `refactor:` Refatoração
- `test:` Testes
- `chore:` Manutenção

### Exemplo:
```
feat: Adiciona filtro de volatilidade ATR

- Implementa cálculo de ATR percentile
- Bloqueia trades quando volatilidade > 80%
- Adiciona testes unitários
```

## ⚠️ Avisos Importantes

### Trading é Arriscado
- Sempre teste em conta DEMO primeiro
- Nunca invista mais do que pode perder
- Este projeto é para fins educacionais
- Não somos responsáveis por perdas financeiras

### Código Sensível
- Nunca commite credenciais (MT5 login, senha)
- Nunca commite tokens (Telegram, APIs)
- Use variáveis de ambiente (.env)
- Adicione arquivos sensíveis ao .gitignore
